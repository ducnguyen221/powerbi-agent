powerbi-agent — AI Agent làm việc trực tiếp với Power BI

MCP server 11 tool + 3 skill (gồm quy trình phân tích KPIM & mẫu tài liệu hóa): truy vấn DAX có chính sách an toàn dữ liệu, dựng trang báo cáo đẹp theo template, và quy trình phân tích end-to-end 9 khâu. Hỗ trợ Claude Code · Codex CLI · Google Antigravity.

2Cài bằng AI Agent — 1 câu lệnh (khuyến nghị)

Dán câu này vào agent của bạn (Claude Code / Codex / Antigravity):

Clone https://github.com/ducnguyen221/powerbi-agent vào ~/.mcp/powerbi-mcp rồi chạy install.ps1 trong đó (đọc script trước khi chạy), sau đó restart MCP host.

Hoặc cài tay:

git clone https://github.com/ducnguyen221/powerbi-agent "%USERPROFILE%\.mcp\powerbi-mcp"
cd "%USERPROFILE%\.mcp\powerbi-mcp"
powershell -ExecutionPolicy Bypass -File .\install.ps1

install.ps1 làm gì (in-place, chạy lại nhiều lần an toàn)

1
Dựng .venv + cài dependencies (rebuild nếu venv copy từ máy khác bị hỏng)
2
Dò ADOMD.NET + TOM đa phiên bản (SSMS 18–22 / standalone / GAC)
3
Đăng ký MCP vào cả 3 host (Claude ~/.claude.json · Codex config.toml · Antigravity mcp_config.json) — đường dẫn suy từ vị trí thư mục, giữ nguyên server khác, backup .bak
4
Copy 3 skill powerbi-mcp + pbi-pipeline + kpim-analysis (kèm references/ + templates/) vào từng host
Tham sốÝ nghĩa
-Hosts claude,codex,antigravityChọn host (mặc định cả ba)
-SkipVenv / -SkipHostsChỉ cập nhật host / chỉ dựng venv
Cài xong phải khởi động lại host (Claude Code / Codex / Antigravity) để tool mới xuất hiện.

5Chạy kèm powerbi-modeling của Microsoft (khuyến nghị)

powerbi-agent không xây lại modeling — phần tạo/sửa hàng loạt table/column/measure/relationship, TMDL, validate DAX giao cho MCP chính chủ Microsoft:

claude mcp add powerbi-modeling -s user -- npx -y "@microsoft/powerbi-modeling-mcp@latest" --start
Phân vai (skill đã dạy agent): query + policy + template/report layer → powerbi-agent · modeling ops → powerbi-modeling. Không ghi 2 server cùng lúc lên 1 model.

6Chính sách an toàn dữ liệu 🛡️

Nguyên tắc: dữ liệu thô ở lại trong engine — chỉ kết quả tổng hợp vào context LLM. Enforce ở server:

LớpMặc địnhĐiều chỉnh
Chặn dump thô (EVALUATE 'Bảng', ALL())BẬTPOWERBI_AGGREGATE_ONLY=0 để tắt
PII blocklist (cột cấm project)Theo fileCopy policy.example.json → policy.json, liệt kê cột
Audit log JSONLBẬT~/.powerbi-agent/audit/ (đổi: POWERBI_AUDIT_DIR)
Trần 200 dòng cho kết quả có cột dimensionBẬTPOWERBI_DIMENSION_ROW_CAP
Trung thực về giới hạn: đây là guard chống rò rỉ sơ ý. Bảo mật cứng = RLS trên model + service principal quyền tối thiểu. Chi tiết kiểm thử: docs/UAT-REPORT.md.

7Power BI Service (cloud — tùy chọn)

1
Tạo Azure Entra ID App Registration, cấp quyền Power BI Service API + add vào workspace
2
Copy .env.example → .env, điền TENANT_ID / CLIENT_ID / CLIENT_SECRET
3
Hỏi agent: "chạy DAX này lên dataset <GUID> trên Service" → execute_dax_service (token có cache)
.env KHÔNG commit, không nằm trong gói chia sẻ — trình cài không bao giờ ghi đè file thật.

8Quy trình KPIM, Template kit & pipeline 9 khâu

Luật sắt: không bao giờ để agent tự dựng layout báo cáo từ đầu — luôn apply_template từ kit đã chứng minh (kit mẫu templates/kpim-business-light đi kèm: card KPI + combo chart + pivot + slicer + map, style đầy đủ).

Trang nào bạn ưng → bảo agent distill_template đóng thành kit riêng — lần sau tái tạo đúng style đó trên dữ liệu khác.

3 skill tự cài vào host:

  • powerbi-mcp — hướng dẫn 11 tool.
  • pbi-pipeline — quy trình kỹ thuật 9 khâu (kết nối → transform M → model → DAX → truy vấn → visual → trang → nâng cao → artifact, mỗi khâu có cổng kiểm) + references/ best-practices DAX/Power Query/SQL (nguồn Microsoft Learn).
  • kpim-analysis — quy trình phân tích KPIM: nhận 1 bộ dữ liệu → tự khảo sát, hỏi ngược người dùng, sinh bộ tài liệu chuẩn hóa (PROJECT.md, DATA_DICTIONARY, METRICS_CALCULATION, DOMAIN_DIMENSION, REPORTS, DESIGN + theme.json, Project_Management.xlsx 6 sheet, 5 mindmap) rồi bàn giao cho pbi-pipeline. Mẫu điền-ngay ở plugins/powerbi-agent/skills/kpim-analysis/templates/.
