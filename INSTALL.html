Power BI MCP Bridge — Hướng dẫn cài đặt

Kết nối Power BI Desktop (cục bộ) & Power BI Service (cloud) vào trợ lý AI Claude Code, Codex và Google Antigravity — chỉ với một lệnh, chạy được trên máy bất kỳ.

Bộ này làm gì? Cài một MCP server Python chạy ngầm, cấp cho AI 3 công cụ truy vấn Power BI bằng DAX/DMV, rồi tự đăng ký vào cả 3 trợ lý AI. Sau khi cài, bạn chỉ cần ra lệnh bằng tiếng Việt tự nhiên — AI tự gọi công cụ, không cần mở terminal.
Windows 10/11 Python ≥ 3.11 Read-only mặc định Không phụ thuộc máy gốc
list_local_reports()
Quét các file .pbix đang mở, trả về port + tên model.
execute_dax_local()
Chạy DAX trực tiếp lên báo cáo Desktop đang mở (qua ADOMD.NET).
execute_dax_service()
Chạy DAX lên dataset đã publish trên cloud (qua REST API + Entra ID).

2Cài đặt — một lệnh (in-place)

Thư mục này vừa là bản chạy, vừa là bộ cài. Mở PowerShell ngay tại thư mục (Shift + chuột phải vào vùng trống → “Open PowerShell window here”) và chạy:

powershell -ExecutionPolicy Bypass -File .\install.ps1

Thế là xong. Trình cài làm ngay tại chỗ (không sinh thư mục thứ hai):

  • Tạo / kiểm tra môi trường Python riêng (.venv) & cài toàn bộ thư viện.
  • Kiểm tra ADOMD.NET.
  • Đăng ký MCP vào cả 3 host, trỏ về chính thư mục này (gộp vào cấu hình có sẵn, không xoá server khác, tự sao lưu .bak).
  • Cài file kỹ năng (SKILL.md) để AI biết cách dùng công cụ.
  • Chạy kiểm thử nhanh.
Mang sang máy mới: copy cả thư mục này (bỏ qua \.venv\ và \.env\), nên đặt vào %USERPROFILE%\.mcp\powerbi-mcp để giống hệt cấu hình gốc, rồi chạy lại install.ps1. .venv sẽ được dựng lại tự động; đường dẫn trong cấu hình host suy theo vị trí thư mục nên không dính username máy cũ.

Tùy chọn nâng cao

# Chỉ đăng ký Claude + Codex
.\install.ps1 -Hosts claude,codex

# Chỉ cập nhật cấu hình host, không đụng venv
.\install.ps1 -SkipVenv

# Chỉ dựng venv, không tự sửa config host nào (tự cấu hình tay)
.\install.ps1 -SkipHosts
Chạy lại install.ps1 nhiều lần là an toàn — nó cập nhật chứ không nhân bản cấu hình.

4Dùng hằng ngày — ví dụ câu lệnh

Bạn nói với AIAI tự làm gì
“Liệt kê các bảng trong báo cáo đang mở.”Chạy EVALUATE INFO.TABLES() qua execute_dax_local.
“Tính tổng doanh thu theo khu vực.”Sinh DAX SUMMARIZECOLUMNS(...) rồi chạy, trả bảng Markdown.

Mặc định kết quả cắt còn 1000 dòng để tránh tràn. AI sẽ tự dùng TOPN/bộ lọc khi khám phá.

5(Tùy chọn) Cấu hình Power BI Service — Cloud

Chỉ cần nếu bạn muốn truy vấn dataset đã publish lên cloud. Mở file .env trong thư mục cài (%USERPROFILE%\.mcp\powerbi-mcp\.env) và điền 3 giá trị từ Azure Entra ID App Registration:

TENANT_ID=your-azure-tenant-guid
CLIENT_ID=your-entra-app-client-guid
CLIENT_SECRET=your-entra-app-client-secret-value
Bảo mật: Không bao giờ commit file .env lên git, không dán nội dung lên chat. Phần Power BI Desktop (local) không cần file này.

6Khắc phục sự cố

❌ “Không tìm thấy Python ≥ 3.11”

Cài Python 3.12 từ python.org, nhớ tick Add to PATH, mở PowerShell mới rồi chạy lại.

❌ Tool local lỗi “FileNotFoundException … AdomdClient.dll”

Máy thiếu ADOMD.NET. Hai cách:

  • Cài “Analysis Services client libraries” từ trang Microsoft (hoặc cài SSMS / Power BI Desktop).
  • Đã có DLL nhưng trình cài không dò ra: mở .env, bỏ comment dòng ADOMD_LIB_DIR= và trỏ tới thư mục chứa Microsoft.AnalysisServices.AdomdClient.dll.

❌ AI không thấy công cụ Power BI

Chưa khởi động lại host. Thoát hẳn trợ lý AI rồi mở lại. Với Claude Code, chạy claude mcp list để kiểm tra.

❌ “list_local_reports” báo không có báo cáo

Power BI Desktop chưa mở file .pbix nào, hoặc model chưa nạp xong. Mở file và đợi vài giây.

Lưu ý port động: Power BI Desktop đổi port mỗi lần mở. Nếu truy vấn lỗi kết nối, bảo AI chạy lại list_local_reports để lấy port mới.
