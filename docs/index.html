Template kit — trang báo cáo đẹp, tái tạo được

Bài học đắt giá: layout agent tự dựng luôn xấu; clone trang đã chứng minh rồi đổi binding thì đẹp. apply_template biến luật đó thành code — style giữ nguyên 100%, chỉ đổi field. Trang nào bạn ưng → distill_template đóng thành kit, dự án sau tái tạo đúng chuẩn đó. Kèm sẵn kit kpim-business-light (card KPI, combo chart, pivot, slicer, map).

Đánh giá thẳng

Điểm mạnh — và giới hạn nói thật

Sản phẩm được UAT trên dashboard production thật (7 trang, 174 measure) — kể cả những gì nó chưa làm được cũng ghi rõ trong UAT report.

Điểm mạnh

  • Report layer mà chưa ai làm — MCP chính chủ Microsoft tuyên bố không sửa trang báo cáo; powerbi-agent dựng/distill trang PBIR bằng clone-and-rebind đã chứng minh bằng sản phẩm thật.
  • Policy enforce ở server — các MCP Power BI khác trả dữ liệu thô tùy agent; ở đây chặn từ tầng code, có audit log làm bằng chứng.
  • Cài 1 lệnh, chạy 3 host — installer idempotent tự dò ADOMD/TOM mọi bản SSMS, đăng ký Claude Code + Codex + Antigravity cùng lúc.
  • Đã UAT trên dữ liệu thật — 17 ca pass trên dashboard production; 5 defect tìm thấy và sửa ngay trong UAT (kể cả bug tồn tại từ code cũ chưa từng chạy live).
  • Vòng tri thức khép kín — trang đẹp thành kit, model thành blueprint; agent giỏi dần theo từng dự án của chính bạn.

Giới hạn (và cách bù)

  • Chỉ Windows — vì Power BI Desktop chỉ có trên Windows. Phần Service (cloud) về lý thuyết chạy đa nền, nhưng chưa hỗ trợ chính thức.
  • Agent không "nhìn thấy" trang render — sau khi dựng trang, người dùng vẫn là người nghiệm thu bằng mắt trong Desktop. Đây là bước bắt buộc trong quy trình.
  • Policy là guard, không phải bảo mật cứng — chặn dump sơ ý bằng phân tích DAX best-effort; dữ liệu nhạy cảm thật sự cần RLS trên model + service principal quyền tối thiểu.
  • Bookmark không tự động hóa — clone bookmark từng làm vỡ báo cáo thật, nên chủ đích để thao tác tay với hướng dẫn từng bước.
  • Ghi trang báo cáo cần file .pbip đóng — ràng buộc của định dạng PBIR; tool tự nhắc trong mọi output.
Cài đặt

Nhanh nhất: đưa cho chính agent cài

Dán 1 câu vào Claude Code / Codex / Antigravity — agent clone, dựng môi trường, đăng ký MCP vào cả 3 host và tự học 4 skill + 6 lệnh /pbi-* đi kèm, cùng bộ references DAX/M/SQL & mẫu tài liệu hóa. Hoặc chạy tay 3 dòng bên cạnh.

  • Windows 10/11 + Power BI Desktop (bản cài hoặc Store)
  • Python 3.11+
  • ADOMD.NET — có sẵn nếu cài SSMS (18–22); installer tự dò mọi vị trí
  • Node.js (tùy chọn) — nếu chạy kèm powerbi-modeling của Microsoft

Hướng dẫn đầy đủ kèm xử lý sự cố: INSTALL.html trong repo.

dán vào agent 
Clone https://github.com/ducnguyen221/powerbi-agent
vào ~/.mcp/powerbi-mcp rồi chạy install.ps1 trong đó
(đọc script trước khi chạy), sau đó restart MCP host.
PowerShell — cài tay 
# 1. Lấy mã nguồn
git clone https://github.com/ducnguyen221/powerbi-agent "$env:USERPROFILE\.mcp\powerbi-mcp"
cd "$env:USERPROFILE\.mcp\powerbi-mcp"
# 2. Cài in-place: venv + dò ADOMD/TOM + đăng ký 3 host + skill
powershell -ExecutionPolicy Bypass -File .\install.ps1
# 3. (khuyến nghị) modeling chính chủ Microsoft chạy kèm
claude mcp add powerbi-modeling -s user -- npx -y "@microsoft/powerbi-modeling-mcp@latest" --start

Nói thật về an toàn dữ liệu: policy của powerbi-agent chặn rò rỉ do sơ ý (agent tiện tay dump bảng) và ghi audit làm bằng chứng — nhưng không thay thế bảo mật cứng. Với dữ liệu nhạy cảm, hãy kết hợp RLS trên model + service principal quyền tối thiểu, và khai báo cột PII vào policy.json ngay đầu dự án.
